# OpenClaw 内容生产 Pipeline

## 架构总览

```
信息采集（自动）→ 选题筛选（人工）→ 内容生成（AI+人工）→ 多平台适配（AI）→ 审核发布（人工）
```

核心原则：AI 干重活，人控质量和方向。

---

## Step 1：信息采集（OpenClaw 自动）

### 配置 OpenClaw 定时监控任务

监控以下信息源，每天汇总一次：

**GitHub**
- openclaw/openclaw 仓库的 Releases、Issues、Discussions
- ClawHub 热门新 Skill（按星标/下载排序）
- GitHub Trending 中 OpenClaw 相关项目

**中文社区**
- 知乎「OpenClaw」话题下的热门问答
- V2EX 相关帖子
- 即刻 OpenClaw 话题

**英文社区**
- Reddit r/openclaw
- Hacker News 相关讨论
- X/Twitter #OpenClaw 热门推文

**新闻/博客**
- 36氪、钛媒体、InfoQ 的 OpenClaw 报道
- 官方博客 openclaw.ai/blog

### 输出格式

OpenClaw 每天早上生成一份「每日情报」，包含：
- 今日热点（3-5条）
- 新发布的值得测评的 Skill
- 社区高赞问题（可以写文章回答的）
- 竞品动态（Kimi Claw、MaxClaw 等）

---

## Step 2：选题筛选（人工 5 分钟）

从每日情报中挑选 1-2 个选题，判断标准：
- 是否有足够多人关心（热度）
- 我能不能写出独特角度（差异化）
- 能不能引导到变现（转化）

---

## Step 3：内容生成（AI 起草 + 人工润色）

### 知乎/公众号长文

用 OpenClaw 生成初稿：
```
提示词模板：

你是一个资深 DevOps 工程师，同时也是 OpenClaw 深度用户。
请围绕「{选题}」写一篇知乎文章。

要求：
- 标题吸引人，带数据或反常识
- 开头用个人经历切入（第一人称）
- 正文包含：问题/背景 → 实操步骤（带代码/截图位置标注）→ 效果/数据 → 总结
- 字数 2000-4000 字
- 文末加引流话术：「更多实战内容，欢迎加入我的 OpenClaw 交流群」
- 语气：专业但不端着，像和朋友聊天
```

人工润色重点：
- 加入真实体验和细节（AI 编不出来的）
- 检查技术准确性
- 调整标题到最吸引人的版本

### 小红书图文

用 OpenClaw 从长文提取要点：
```
提示词模板：

将以下文章改写为小红书风格的图文笔记。

要求：
- 标题：带 emoji，20字以内，制造好奇心
- 正文：分 3-5 个要点，每个要点 1-2 句话
- 标注需要配图的位置（截图/示意图）
- 结尾加话题标签：#OpenClaw #AI #效率工具
- 语气：轻松、实用、像分享给朋友
```

### B站视频脚本（5-15分钟中长视频）

```
提示词模板：

将以下文章改写为 B站教程视频脚本（5-10分钟）。

要求：
- 开头 15 秒抓住注意力（提出问题或展示结果）
- 正文分步骤讲解，标注需要录屏演示的位置
- 每 2-3 分钟设置一个小高潮或转折
- 结尾总结 + 引导关注/评论
- 语气：自然口语化，不要念稿感
```

### 抖音短视频脚本（30秒-1分钟）

```
提示词模板：

将以下文章提炼为抖音短视频脚本（30-60秒）。

要求：
- 前 3 秒必须有钩子（一句话制造冲突或好奇："OpenClaw 一天烧400块？"）
- 正文只讲 1 个核心点，不要贪多
- 每句话都要短，适合口播节奏
- 结尾引导互动："你们觉得值不值？评论区聊"
- 标注是否需要录屏画面
- 语气：直接、有能量、不啰嗦
```

## Step 4：多平台适配（AI 自动）

一篇内容生成后，OpenClaw 自动适配为 7 个平台：

| 平台 | 格式 | 特殊要求 |
|------|------|---------|
| 知乎 | Markdown 长文 | 加粗重点，代码块格式化 |
| 公众号 | 富文本 | 需要排版（用 Markdown Nice 转换）|
| 小红书 | 短图文 | 需要配图，文字精简 |
| B站 | 中长视频脚本（5-15分钟）| 标注录屏点 |
| 抖音 | 短视频脚本（30-60秒）| 钩子开头，只讲1个点 |
| X/即刻 | 短帖 | 140字以内，带链接 |
| YouTube | 英文脚本 | AI 翻译+润色 |

### 视频内容生产方式（可选）

根据情况灵活选择：

| 方式 | 适用场景 | 工具 |
|------|---------|------|
| **真人口播 + 录屏** | 抖音/B站，效果最好 | 手机/摄像头 + OBS |
| **纯录屏 + 真人配音** | 教程类，不想露脸时 | OBS + 麦克风 |
| **纯录屏 + AI 配音** | 批量产出，省时间 | OBS + 剪映 AI 朗读 |

建议：抖音优先真人口播（算法加权），B站录屏+真人配音即可。

### 一鱼多吃：视频内容复用

```
录制一次完整操作演示（10-15分钟）
  │
  ├── 完整版 → B站 / YouTube
  │
  ├── 截取精华片段（30-60秒）→ 抖音
  │   可截取多个片段，一条视频变 3-5 条抖音
  │
  └── 关键帧截图 → 小红书配图
```

## Step 5：审核发布（人工）

**必须人工做的事**：
- 检查事实准确性（AI 可能编造数据）
- 检查是否有敏感内容（政策/法规相关）
- 最终决定发布时间
- 回复评论互动（前期手动，后期可半自动）

**发布时间建议**：
- 知乎：工作日 20:00-22:00
- 小红书：12:00-13:00 / 20:00-22:00
- B站：18:00-20:00（周末更佳）
- 抖音：12:00-13:00 / 18:00-20:00 / 21:00-22:00
- 公众号：07:30-08:30 / 20:00-21:00

## 工具链

| 环节 | 工具 |
|------|------|
| 信息采集 | OpenClaw + RSS Skills + Browser Skill |
| 内容生成 | OpenClaw（Claude/GPT 作为底层模型）|
| 图片制作 | OpenClaw + anzai-image-gen Skill / Canva |
| 视频录制 | OBS（录屏）+ 剪映/DaVinci（剪辑）|
| 排版 | Markdown Nice（公众号）/ 小红书原生编辑器 |
| 发布 | 人工发布（避免自动发布被风控）|
| 数据追踪 | 各平台原生数据后台 |
